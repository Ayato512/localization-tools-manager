# 本地化工具箱

一组本地化 / 翻译技术工作中常用的小工具集合。

## 🚀 怎么用？

1. **第一次使用前**：双击 `首次安装.bat`（Windows）或 `首次安装.command`（Mac），安装依赖。
2. **平时使用**：双击 `启动器.py`，点想用的工具按钮，即可弹出对应工具窗口。

> 如果双击 `启动器.py` 没反应，可能是 Python 没设置默认打开方式。可以右键 → 打开方式 → 选 Python，或者在 VS Code 里打开本文件夹按 F5 启动。

## 🧰 工具清单

| 工具 | 用途 | 备注 |
|------|------|------|
| 🔍 **Excel 搜索器** | 在多个 Excel 文件 / 多个 Sheet 里跨表搜索内容 | ⚠️ 仅 Windows |
| 📝 **语料对齐** | 中日文语料按标点同步拆分对齐（含模糊匹配） | |
| 🔄 **TMX 转换** | 把 Excel 双语对照表转成翻译记忆库 TMX 格式 | |
| 📚 **术语抽取** | 从中文语料里基于熵 + PMI 自动抽取术语候选 | |
| 📊 **Excel 合并** | 把多个 Excel 文件按原样合并到一个工作簿 | |
| ✂️ **Excel 拆分** | 把一个多 Sheet 的 Excel 拆成多个独立文件（保留格式/图片/公式） | |
| 📐 **Excel 重排对齐** | Excel 列重排、按下划线尾号对齐 | |
| 🎨 **美术资产对照** | 调用 Gemini 模型对美术资产做 OCR / 对照比较 | 需要 API key |

## 📁 文件夹是干啥的？

```
modmanager/
├── 使用说明.md              ← 你正在看的文件
├── 启动器.py                ← 双击这个启动菜单
├── 首次安装.bat / .command  ← 第一次使用前装依赖
├── requirements.txt         ← 依赖清单（给 pip 看的）
├── tools/                   ← 真正的工具脚本都在这里
├── assets/                  ← 资源文件夹（图标等）
├── 日志/                    ← 出错日志会放这里（自动生成）
└── .vscode/                 ← VS Code 配置（自动生效）
```

## 💾 关于版本管理（改代码后留个版本）

老版本的 `modmanager` 用 `.bak` 文件做版本备份，**现在改用 Git + GitHub**。好处：能看每次改了什么、可以远程同步、可以回到任意历史版本。

**日常操作（在 VS Code 里点鼠标，不用命令行）**：

1. 改完代码 → VS Code 左侧第三个图标（🌿 分叉图标）= **源代码管理**
2. 看到改过的文件后面带 "M" 标记
3. 在上方输入框写一句话说明（比如"修复 TMX 转换日期格式"）
4. 点 ✓ 提交（commit）
5. 点 "..." → "推送"（push），同步到 GitHub

**想看某个文件的历史版本**：

- 装 GitLens 插件（VS Code 会自动推荐）
- 文件标签页右键 → "Open File History"
- 或者直接在 GitHub 网页上看

**想回到老版本**：

- 在 VS Code 的文件历史里找到老版本，点击即可查看
- 复制内容覆盖回来再 commit 一次就行

## ⚠️ 注意事项

- 代码里如果有 API key（比如美术资产对照工具的 Gemini key），**提交前自己看一眼别 commit 上去**——尤其是 GitHub 公开仓库
- `日志/` 里的 .log、`__pycache__/`、`.bak` 文件已经在 `.gitignore` 里排除了，不会被提交
- 想新加一个工具？直接把 `.py` 文件丢进 `tools/`，然后在 `启动器.py` 顶部的 `TOOLS` 列表加一行即可

## 🆘 出错了怎么办？

1. 先看 `日志/` 文件夹下最新的 .log 文件
2. 看 `excel_tool_error.log`（Excel 工具会把错误写这里）
3. VS Code 里直接 F5 跑工具，能看完整报错
4. 找不到原因？把报错截图 + 操作步骤发出来求助

## 🛠️ 开发环境（推荐）

- **VS Code**——打开本文件夹会自动加载 `.vscode/` 里的配置
- **推荐插件**（VS Code 会主动弹窗推荐）：
  - Python（必装）
  - Pylance（必装）
  - GitLens（看文件历史很爽）
- **Python 版本**：3.10 或更高
